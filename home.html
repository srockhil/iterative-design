Introduction

In this project we set out to learn about iterative design, and how this process of prototyping, user-testing and analyzing is essential to the development of a good user interface.

The Startup: Ribbon

Ribbon gives creators and businesses a way to sell tickets for online virtual events. The service facilitates payment, invitations, and zoom-link creation, allowing creators to focusing on making their online experience great for their customers.

Hi-Fi Mockup

After finalizing our wireframes, we created a hi-fi mockup of the interface. The following details this process and the choices made, as well as how we incorporated feedback from student critiques.

Startup Goals Prioritized in Design:

  • Prioritization of easy and fast event creation
  • Ability for users of all technology skill levels to easily navigate site
  • Quick and intuitive Zoom integration
  • Readily accessible stats on events
  • Detailed event customization options (ticket price, etc.)

Implementation of Usability Principles:

  • Visually similar elements represent similar objects across the site, improving learnability.
  • Unified font styles and card layouts creates hierarchy.
  • Navigation has been made simple via a bar at the top and previous and future buttons on events page.
  • Event preview screen improves efficiency.
  • Vertical form with certain options graying out and "or" options being side by side improves hierarchy and learnability.

Feedback Incorporation:


Critique Implemented Solution
The plus symbol is used to both view more information and add a new event. Replaced the plus symbol used for viewing more information with a downward arrow. This differentiates the two actions and motivates the sliding down animation
Unclear distinction between past and future events Added past and future event pagination buttons
No way to find events/filter events No way to find events/filter events
Inconsistent form containers Standardized our containers and used similar models across different areas of the form.
Lack of text hierarchy Changed the text sizes so there was a clear distinction between header, subheader, and paragraph.
Duration field is unclear what it refers to Used hrs and mins to make imputs clear
Big font and block sizes make the page look like a tablet Decrease block and font sizes
Monthly stats are not detailed and don't portray timeline for when stats refer to Added details of what stats might consist of, as well as toggle for different timelines

The Mockup

User Testing

Lastly we rolled out user tests to test the usability and learnability of our site on real users

Tasks and Subtasks

  1. Create a new event that reoccurs every Tuesday
  2. Find the number of participants coming to your upcoming event.
  3. Locate the bank account information associated with the Ribbon account.
  4. View all the participants attending one of your upcoming event.

Results and Analysis

Task Result Analysis
Create a new event that reoccurs every Tuesday Some users were confused by the choice between repeat every week or repeat monthly. We were surprised by this because we believed the "or" text was sufficient indicating the either/or between the two
Find the number of participants coming to your upcoming event All users did this efficiently, and claimed this task was very "easy", although some were slower at finding the button We believe this confusion stemmed from inconsistent language in using attendees and participants synonymously.
Locate the bank account information associated with the Ribbon account Two users found this information after some time, one user didn't understand the concept of a user's bank information, and performed no actions This was surprising to us, as we assumed that profile would be the only place one would look for bank information. However, we believe that the word "profile" might indicate an individual user instead of an organization, which might have confused our testers.
View all the participants attending one of your upcoming events. All users intuitively found "all participants", and no user discussed any difficulty with this task. We were not surprised as to the ease with which users found this information, as we made a point in making the button explicit.
