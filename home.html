Introduction

In this project we set out to learn about iterative design, and how this process of prototyping, user-testing and analyzing is essential to the development of a good user interface. To do this, we undertook the task of designing an interface for a startup: Ribbon. We started by reading about the company to gain insight into their purpose and important features they offer. Using this knowledge we made initial sketches and used them to create lo-fi wireframes of our design. From there we created our first hi-fi mockup. This mockup was critiqued by our classmates, and we used the feedback to improve our interface before submitting it to a user testing site. Finally, we analyzed the results of 3 usersâ€™ questionnaires and video-responses.

The Startup: Ribbon

Ribbon gives creators and businesses a way to sell tickets for online virtual events. The website facilitates everything from event creation to ticket sales. Users can create weekly, monthly or any type of subscription to events. Their zoom integration automatically creates meetings with unique links and passwords for each event. The site allows users to manage and edit their events, see event participants, and quickly invite new event participants.
